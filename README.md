# gesture-app

Tiny background macOS listener for two trackpad gestures:

- **Hold one finger, tap to its right** → next tab (`Ctrl+Tab`)
- **Hold one finger, tap to its left** → previous tab (`Ctrl+Shift+Tab`)

No window, no menu bar icon. It just listens and posts keystrokes.

## How it works

- Reads raw per-finger trackpad positions from Apple's private
  `MultitouchSupport.framework` (the public `NSTouch` API can't see touches
  globally). **Private API — personal use only, not App Store safe.**
- First finger down = the *anchor*. A second finger that taps and lifts quickly
  while the anchor stays down triggers the switch, based on which side of the
  anchor it tapped.
- Two-finger taps, scrolls, and drags are ignored (the anchor lifts or both
  fingers stay down, so nothing fires).

## Build

```sh
swift build -c release
```

Binary: `.build/release/gesture-app`

## Run

### App bundle (recommended)

```sh
./make-icon.sh     # generate AppIcon.icns (once; regenerate to change art)
./build-app.sh     # build + bundle + sign
open GestureTabs.app
```

The icon is generated by `tools/make-icon.swift` (a trackpad with a held
anchor dot and left/right chevrons) — no external assets. `build-app.sh` folds
`AppIcon.icns` into the bundle if present.

Produces `GestureTabs.app` — a menu-bar agent (no dock icon) holding both
binaries. The `●`/`○` menu item and Start/Stop/Quit work as below. The bundle is
ad-hoc code-signed so the Accessibility grant sticks to one app (`GestureTabs`).

Move it wherever you like (e.g. `/Applications`); both binaries live inside, so
it's self-contained.

**Start at login:** System Settings → General → Login Items → add
`GestureTabs.app`.

**Note:** re-running `build-app.sh` re-signs ad-hoc, which can make macOS ask you
to re-grant Accessibility after a rebuild.

## Run (dev, unbundled)

Two ways.

### Menu-bar controller (recommended)

```sh
./.build/release/gesture-control
```

Adds a menu-bar item (no dock icon):

- `●` = service running, `○` = stopped
- **Start** / **Stop** — launch or kill the background `gesture-app` service
- **Quit** — stops the service and exits

It auto-starts the service on launch and finds `gesture-app` sitting next to it,
so keep both binaries in the same folder.

### Service directly

```sh
./.build/release/gesture-app          # foreground, Ctrl+C to stop
./.build/release/gesture-app &        # detached
```

## Required permission

Posting keystrokes needs **Accessibility** permission for whatever process runs
the binary (your terminal, or the binary itself):

**System Settings → Privacy & Security → Accessibility** → enable the terminal
app (or the `gesture-app` binary).

If it isn't granted, the app still runs and detects gestures but prints a
warning and can't switch tabs.

## Tuning

Edit the constants at the top of `Sources/gesture-app/main.swift`:

- `sideThreshold` — how far to the side the tap must be to count.
- `tapMaxDuration` — how quick a tap must be.
- `kVK_Tab` / the flags in `switchTab` — change the emitted keystroke.
